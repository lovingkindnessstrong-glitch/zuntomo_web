# Task: トップページ（`/`）のAstroマイグレーション

## 1. 目的
既存のトップページ（HTML/CSS）のコードを流用し、見た目やデザインを一切変更することなく、Astroのページ（`src/pages/index.astro`）およびレイアウトコンポーネントとして移植・再構築する。

## 2. 前提条件
作業開始前に、必ずプロジェクトルートにある `project_guidelines.md` を熟読し、プロジェクトのルーティング要件、デザインシステム、およびAIに対する厳守事項（Technology Stack & Migration Rules）を把握すること。

## 3. 実装ステップ
1. **現状分析:** 既存のトップページに該当する静的HTMLファイルと、適用されているCSSファイル（CSS変数の定義を含む）を読み込み、構造を解析する。
2. **ベースレイアウトの切り出し:** 全ページ共通となる `<head>` 情報、ヘッダー、フッターなどを抽出し、`src/layouts/Layout.astro`（またはプロジェクトに最適なレイアウトファイル）を作成する。
3. **トップページの実装:** トップページ固有のコンテンツブロックを抽出し、`src/pages/index.astro` を構築する。
4. **パスの解決:** 画像（ロゴや写真など）やCSSファイルへのパスを、Astroのディレクトリ構造（`public` や `src/assets` など）に合わせて正しく解決する。

## 4. 厳守事項（AI Agentへの警告）
- **既存デザインの完全保持:** 既存のHTMLタグ構造、CSSクラス名、変数は「そのまま」流用すること。AIによる独自のUI改善、レイアウト変更、リファクタリングは**一切禁止**する。
- **コンポーネント化の粒度:** 初期段階では細かすぎるコンポーネント分割（ボタン単体の切り出しなど）は避け、まずはページ全体がAstro上でエラーなく描画されることを最優先とする。

## 5. 完了条件（DoD）
- ターミナルでAstroの開発サーバーを起動し、トップページがエラーなく表示されること。
- 既存の静的HTMLと、Astro化されたページの間で、視覚的な差分（デザイン崩れ）が一切発生していないこと。
- 全ての作業と確認が完了した後、このタスクファイル自身を `docs/archive/` ディレクトリに移動させること。
